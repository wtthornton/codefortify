Installation Guide

This guide covers all the different ways to install CodeFortify on your system. Choose the method that best fits your development workflow.

üí° Pro Tip: We recommend the global npm installation for the best experience, but local installation works great for team projects.

Prerequisites

Before installing CodeFortify, make sure you have the following installed:

Required Software

  • Node.js: Version 18.0.0 or higher
  • npm: Version 8.0.0 or higher (comes with Node.js)
  • Git: For version control integration

Method 1: Global Installation (Recommended)

Install CodeFortify globally to use it from anywhere on your system:

# Install globally
npm install -g codefortify

# Verify installation
codefortify --version
‚úÖ Benefits: Use CodeFortify in any project directory, consistent version across projects, easy updates.

Global Installation Steps

Step 1: Check Node.js Version

Ensure you have Node.js 18.0.0 or higher:

node --version
npm --version

If you need to update Node.js, visit nodejs.org

Step 2: Install CodeFortify

Run the global installation command:

npm install -g codefortify

This may take a few minutes as it downloads and installs all dependencies.

Step 3: Verify Installation

Check that CodeFortify is properly installed:

codefortify --version
codefortify --help

You should see the version number and help information.

Step 4: Test in a Project

Navigate to any project directory and test:

cd my-project
codefortify validate

This will analyze your project and show the current state.

Method 2: Local Installation

Install CodeFortify as a development dependency in your project:

# Navigate to your project
cd my-project

# Install as dev dependency
npm install codefortify --save-dev

# Use via npx
npx codefortify --version
‚ö†Ô∏è Note: With local installation, you'll need to use npx codefortify instead of just codefortify.

Local Installation Benefits

  • Version Control: Each project can use a specific version
  • Team Consistency: All team members use the same version
  • CI/CD Integration: Easy to include in build pipelines
  • No Global Dependencies: Keeps your system clean

Method 3: Docker Installation

Run CodeFortify in a containerized environment:

# Pull the Docker image
docker pull codefortify/cli:latest

# Run CodeFortify
docker run -v $(pwd):/workspace codefortify/cli score

Docker Benefits

  • Isolation: No system dependencies
  • Consistency: Same environment everywhere
  • CI/CD Ready: Easy integration with pipelines
  • Version Control: Pin specific versions

Method 4: Development Installation

For contributors and advanced users who want to build from source:

# Clone the repository
git clone https://github.com/codefortify/context7-mcp.git
cd context7-mcp

# Install dependencies
npm install

# Build the project
npm run build

# Link globally for development
npm link
üîß Development Mode: This gives you access to the latest features and allows you to contribute to the project.

Platform-Specific Instructions

Windows

Windows Installation

  1. Download and install Node.js from nodejs.org
  2. Open PowerShell or Command Prompt as Administrator
  3. Run: npm install -g codefortify
  4. Verify: codefortify --version

macOS

macOS Installation

  1. Install Node.js using Homebrew: brew install node
  2. Or download from nodejs.org
  3. Run: npm install -g codefortify
  4. Verify: codefortify --version

Linux

Linux Installation

  1. Install Node.js using your package manager:
    • Ubuntu/Debian: sudo apt install nodejs npm
    • CentOS/RHEL: sudo yum install nodejs npm
    • Arch: sudo pacman -S nodejs npm
  2. Run: npm install -g codefortify
  3. Verify: codefortify --version

Verification and Testing

After installation, verify everything is working correctly:

# Check version
codefortify --version

# View help
codefortify --help

# Test with a sample project
mkdir test-project
cd test-project
echo "console.log('Hello World');" > index.js
codefortify validate

Troubleshooting

Common installation issues and solutions:

Permission Errors (macOS/Linux)

If you get permission errors, try using sudo or fix npm permissions:

# Option 1: Use sudo (not recommended)
sudo npm install -g codefortify

# Option 2: Fix npm permissions (recommended)
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH
npm install -g codefortify

Command Not Found

If the codefortify command is not found:

  • Check your PATH environment variable
  • Restart your terminal/command prompt
  • Try using the full path: /usr/local/bin/codefortify

Node.js Version Issues

If you have Node.js version conflicts:

  • Use nvm to manage Node.js versions
  • Update to Node.js 18.0.0 or higher
  • Clear npm cache: npm cache clean --force

Updating CodeFortify

Keep CodeFortify up to date with the latest features and improvements:

# Update global installation
npm update -g codefortify

# Check for updates
npm outdated -g codefortify

# Update local installation
npm update codefortify

Uninstalling CodeFortify

If you need to remove CodeFortify:

# Uninstall global installation
npm uninstall -g codefortify

# Uninstall local installation
npm uninstall codefortify
