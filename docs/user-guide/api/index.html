CLI Commands Reference

Core Commands

codefortify validate [options]

Validates project structure and configuration

Options:
OptionDescriptionDefault
--detailedShow detailed validation resultsfalse
--config <file>Use custom config file.codefortify.json
--fixAuto-fix common issuesfalse
--jsonOutput in JSON formatfalse
Examples:
# Basic validation
codefortify validate

# Detailed validation with fixes
codefortify validate --detailed --fix

# Validate with custom config
codefortify validate --config production.json

codefortify score [options]

Analyzes code quality and provides scoring

Options:
OptionDescriptionDefault
--detailedShow detailed score breakdownfalse
--files <pattern>Score specific filesall
--threshold <score>Set quality threshold80
--report <file>Generate report filenone
Examples:
# Basic scoring
codefortify score

# Score specific files with detailed output
codefortify score --files "src/**/*.tsx" --detailed

# Generate HTML report
codefortify score --report quality-report.html

codefortify enhance [options]

Applies AI-powered code improvements

Options:
OptionDescriptionDefault
--files <pattern>Enhance specific filesall
--focus <area>Focus on specific areaall
--dry-runPreview changes without applyingfalse
--interactiveInteractive enhancement modefalse
--continuousEnable continuous enhancementfalse
Focus Areas:
  • performance - Optimize for performance
  • security - Apply security improvements
  • readability - Improve code readability
  • maintainability - Enhance maintainability
  • testing - Add or improve tests
Examples:
# Basic enhancement
codefortify enhance

# Performance-focused enhancement
codefortify enhance --focus performance

# Interactive mode with dry run
codefortify enhance --interactive --dry-run

codefortify monitor [options]

Starts real-time monitoring of codebase

Options:
OptionDescriptionDefault
--interval <time>Check interval30s
--watch <paths>Directories to watchsrc,lib
--daemonRun in backgroundfalse
--dashboardEnable web dashboardfalse
--notifyEnable notificationsfalse
Examples:
# Basic monitoring
codefortify monitor

# Monitor with dashboard
codefortify monitor --dashboard --interval 60s

# Background monitoring
codefortify monitor --daemon --notify

Utility Commands

codefortify status [options]

Shows system and project status

Options:
  • --detailed - Show detailed status information
  • --health - Include health check results
  • --components <list> - Check specific components
  • --export <file> - Export status to file

codefortify logs [options]

View and manage system logs

Options:
  • --level <level> - Filter by log level (debug, info, warn, error)
  • --follow - Follow logs in real-time
  • --component <name> - Filter by component
  • --export <file> - Export logs to file

codefortify health [options]

Performs comprehensive health checks

Options:
  • --detailed - Show detailed health information
  • --fix - Attempt to fix issues automatically
  • --components <list> - Check specific components
  • --export <file> - Export health report

Programmatic API

Basic Usage

Installation

npm install codefortify

Basic API Usage

const CodeFortify = require('codefortify');

// Initialize CodeFortify
const cf = new CodeFortify({
  config: './codefortify.json'
});

// Validate project
const validation = await cf.validate();
console.log('Validation result:', validation);

// Score code quality
const score = await cf.score();
console.log('Quality score:', score.overall);

// Enhance code
const enhancement = await cf.enhance({
  focus: 'performance',
  files: ['src/**/*.tsx']
});
console.log('Enhancement result:', enhancement);

API Classes

CodeFortify Class

Main class for CodeFortify operations

Constructor Options:
  • config - Path to configuration file
  • cwd - Working directory
  • verbose - Enable verbose logging
Methods:
  • validate(options) - Validate project
  • score(options) - Score code quality
  • enhance(options) - Enhance code
  • monitor(options) - Start monitoring
  • status() - Get system status

EnhancementAgent Class

Direct access to enhancement agent

const { EnhancementAgent } = require('codefortify');

const agent = new EnhancementAgent({
  focus: 'performance',
  approvalRequired: true
});

const result = await agent.enhance('src/components/Button.tsx');
console.log('Enhancement applied:', result.success);

Configuration Reference

Configuration File (.codefortify.json)

{
  "version": "1.0.0",
  "monitoring": {
    "enabled": true,
    "interval": "30s",
    "watchPaths": ["src", "lib"],
    "ignorePaths": ["node_modules", "dist"]

  "enhancement": {
    "autoApply": false,

  }
}

Integration Examples

CI/CD Integration

# GitHub Actions example
name: CodeFortify Quality Check
on: [push, pull_request]

jobs:
  quality-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Setup Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '18'
      - name: Install CodeFortify
        run: npm install -g codefortify
      - name: Validate Project
        run: codefortify validate
      - name: Score Quality
        run: codefortify score --threshold 80
      - name: Generate Report
        run: codefortify score --report quality-report.html

VS Code Extension Integration

// VS Code extension example
const vscode = require('vscode');
const { CodeFortify } = require('codefortify');

class CodeFortifyProvider {
  constructor() {
    this.cf = new CodeFortify();
  }

  async provideCodeActions(document, range) {
    const score = await this.cf.score({
      files: [document.fileName]
    });

    if (score.overall < 80) {
      return [{
        title: 'Enhance with CodeFortify',
        command: 'codefortify.enhance',
        arguments: [document.fileName]
      }];
    }
  }
}
